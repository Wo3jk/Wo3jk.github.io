内网渗透实验记录

场景：通过 Windows 2003 跳板机 渗透仅内网可达的 Windows 2008，实践内网横向移动与端口转发技术。

第一阶段：实验环境搭建（网络拓扑）

为模拟“无法直接访问内网”的真实场景，使用 VMware 虚拟网络 进行隔离。

1. 攻击机（Kali Linux）

  • 网卡1：NAT
  • IP（示例）：192.168.59.128

2. 跳板机（Windows 2003）

  • 网卡1：NAT（对外）

    • IP（示例）：192.168.59.135

  • 网卡2：Host-only（内网）

    • IP（示例）：10.10.10.10

  • 关键配置：

    • 安装 IIS 6.0
    • 在 Web 服务扩展 中 启用 WebDAV

3. 内网目标（Windows 2008）

  • 网卡1：Host-only
  • IP（示例）：10.10.10.20
  • 仅允许 Windows 2003 访问

第二阶段：突破边界（攻击 Windows 2003）

利用漏洞：CVE-2017-7269（IIS 6.0 WebDAV RCE）

⚠️ 问题：IIS 非常不稳定，多次尝试后 IIS 服务直接崩溃，攻击失败。

备用方案：MS08-067





由于 Windows 2003 版本过旧，在构造 Payload 时直接使用 Meterpreter，避免后续升级失败。

1. SMB 存活扫描

2. MS17-010 漏洞检测

返回 VULNERABLE，确认存在 永恒之蓝漏洞。

第六阶段：横向移动（攻击 Windows 2008）

尝试 1：EternalBlue（失败）

❌ 结果：

  • SMB 服务直接崩溃
  • 出现 IOError / closed stream

原因分析：

  • EternalBlue 对时序极其敏感
  • Autoroute 转发存在延迟
  • 内核内存 Grooming 失败，导致 Windows 2008 蓝屏

第七阶段：使用 Portfwd 绕过 Autoroute 不稳定性（关键）

1. 清理现场

  • 重启 Windows 2008（必须）
  • 停止 MSF 所有后台任务

原因总结

  • 永恒之蓝 数据流量巨大
  • 通过跳板转发后 数据包时序紊乱
  • Windows 2008 SMB 进程频繁崩溃
  • 靶机过于脆弱，几乎“一打就死”

实验核心知识点总结

  • CVE-2017-7269：IIS 6.0 WebDAV 经典 RCE，内网入场券

  • MS08-067：老系统兜底神器

  • Autoroute：MSF 虚拟路由机制

  • Portfwd：稳定、专业的内网隧道手段

  • MS17-010（EternalBlue）：

    • 对网络时延极其敏感
    • 在跳板环境下成功率很低

总结一句话

内网渗透最难的不是拿权限，而是如何让“不稳定的漏洞”在“不稳定的网络”中稳定执行。

这次实验虽然没能稳定拿下 Win2008，但完整走通了 真实内网攻击思路，非常有价值。
