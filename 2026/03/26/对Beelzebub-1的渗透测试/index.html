对Beelzebub-1的渗透测试

发布于 2026-03-26 分类：内网渗透, 靶机日记 最后更新于 2026-04-01 👁

使用nmap主机发现靶机ip192.168.59.153
nmap -sP 192.168.59.0/24

使用nmap端口扫描发现靶机开放端口：22、80
nmap -sV -p- 192.168.59.153

80端口没什么东西，扫一下目录
dirsearch -u 192.168.59.153


对beelzebub进行md5编码d18e1e22becbd915b45e0e655429d487，把这个拼接到URL后面，没有东西，再扫一下。

有目录遍历


有一个界面不知道干啥的，我们随便输一点东西

cookie里面存在一个密码，我们枚举一下用户名
wpscan --url http://192.168.59.153/d18e1e22becbd915b45e0e655429d487/ -e u --ignore-main-redirect --force

3参数 作用
–url http://192.168.59.153/d18e1e22becbd915b45e0e655429d487/ 指定目标 WordPress 网站的 URL（可能是本地测试环境或内网 IP）。
-e u 枚举用户 (User Enumeration)，尝试获取 WordPress 网站上的用户名列表。
–ignore-main-redirect 忽略主重定向，防止 WPScan 因网站重定向（如 301/302）而停止扫描。
–force 强制扫描，即使目标网站有反爬机制（如 WAF、速率限制），仍然继续扫描。
扫到两个用户名，使用ssh尝试连接

提权

方法一（SUID）

find / -perm -u=s -type f 2>/dev/null

这个pkexec存在漏洞，提权方法见[[对jangow-1.0.1的渗透测试)

方法二（Serv-U 提权）


家目录下存在Serv-U的配置文件，运行着Serv-U服务，该程序的Serv-U-Tray组件存在权限提升漏洞
查看.bash_history文件


跟着命令执行一遍就变成root了

方法三（内核提权）

uname -a

搜了一下这个内核版本存在提权漏洞

CVE-2021-3493 (Ubuntu OverlayFS)

这是一个非常经典且稳定的 Ubuntu 提权漏洞，影响 18.04/20.04 等多个版本。
在github中找到对应的exp，复制到攻击机中，开启http服务


靶机下载编译运行，拿到root权限
